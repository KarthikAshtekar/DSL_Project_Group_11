# DSL Project Final Notebooks

This repository contains the final notebook-style Python scripts used to train the models, generate the forecast outputs, and visualise the technician allocation project.

## What is in `Final_notebooks/`

- `notebook1_rnn_lstm_forecasting.py`
  - Trains the BiRNN and BiLSTM severity classifiers
  - Builds the time-series forecast data
  - Saves the final forecast Excel file used by the dashboard
- `notebook2_bert.py`
  - Fine-tunes a BERT severity classifier
  - Saves the trained model and confusion matrix plot
- `notebook3_dashboard.py`
  - Runs the Streamlit dashboard
  - Uses the forecast Excel file to optimise technician allocation and show the visual analytics

## Recommended run order

1. Run Notebook 1 first to generate the forecast artifacts.
2. Run Notebook 2 if you want the BERT-based severity classifier outputs.
3. Run Notebook 3 to open the interactive dashboard and visualise the final project.

## Requirements

- Python 3.10 or later is recommended
- A virtual environment is strongly recommended
- For Notebook 3, a valid Gurobi license is required
- Internet access may be needed the first time for model downloads and package installs

## Install dependencies

Create and activate a virtual environment, then install the packages used by the final notebooks:

```bash
pip install pandas numpy matplotlib seaborn scikit-learn statsmodels tensorflow openpyxl streamlit transformers datasets torch scipy gurobipy
```

If you prefer Jupyter or Colab, you can install the same packages there before running the scripts.

## Data files needed

The notebooks expect the project data to be available locally. The main input used by the final pipeline is:

- `data/raw/Train_TEST.csv`

Notebook 1 and Notebook 2 reference `Train_TEST.csv` directly in their code. The easiest way to run them locally is to either:

- copy `Train_TEST.csv` into the same working directory before running the notebook, or
- update the `TRAIN_CSV` path in the script to point to `data/raw/Train_TEST.csv`

## How to run Notebook 1

Notebook 1 should be run top to bottom.

### In Colab

1. Open `Final_notebooks/notebook1_rnn_lstm_forecasting.py`
2. Upload `Train_TEST.csv`
3. Run the file top to bottom

### Locally

1. Open the file in VS Code, Jupyter, or another notebook-friendly editor
2. Make sure `TRAIN_CSV` points to the correct CSV file
3. Run the script from top to bottom

### Outputs generated by Notebook 1

- `bilstm_birnn_model.h5`
- `bilstm_model_improved.h5`
- `tokenizer.pkl`
- `all_plots.pdf`
- `seasonal_index_data.csv`
- `FINAL_forecast_results.xlsx`

The file `FINAL_forecast_results.xlsx` is the key input for the dashboard in Notebook 3.

## How to run Notebook 2

Run Notebook 2 after the dataset is available.

```bash
python Final_notebooks/notebook2_bert.py
```

### Outputs generated by Notebook 2

- `bert_severity_model/`
- `bert_label_map.pkl`
- `bert_confusion_matrix.png`

## How to run Notebook 3

Notebook 3 is a Streamlit app.

```bash
streamlit run Final_notebooks/notebook3_dashboard.py
```

When the app opens:

1. Upload `FINAL_forecast_results.xlsx` in the sidebar
2. Click `Run 30-Day Optimization`
3. Review the dashboard charts, summaries, and allocation results

## Visualisation workflow

If your goal is only to visualise the final project, the shortest path is:

1. Run Notebook 1
2. Upload the resulting `FINAL_forecast_results.xlsx` into Notebook 3
3. Use the dashboard to inspect the allocation results and charts

If you also want the NLP severity-classification visuals, run Notebook 2 as well.

## Notes

- Notebook 1 is the most important prerequisite because it generates the forecast file used by the dashboard.
- Notebook 3 will not run correctly unless the forecast Excel file has the expected columns.
- Gurobi is required only for the optimisation step in Notebook 3.
